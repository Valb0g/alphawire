# Alpha Posts — Двухуровневая система постов

## Проблема

Канал выглядит как безжизненный автомат по выдаче новостей. Все посты одинаковые по формату и тону — сухие, нейтральные, без характера. Нет причин для аудитории обсуждать контент.

## Решение

Двухуровневая система постов, где **обычные новости** идут в текущем формате ленты, а **альфа-новости** (score ≥ 9) получают авторский формат с характером, вопросом к аудитории и кнопкой для перехода к обсуждению.

## Дизайн

### Обычный пост (score 6–8) — без изменений

```
🔵 Whale перевёл 15,000 ETH на Binance

Крупный держатель переместил 15,000 ETH ($38M) на Binance.
Транзакция может сигнализировать о намерении продать.

🔗 LookOnChain
```

### Альфа-пост (score ≥ 9) — новый формат

```
🚨🔴 Взломан мост Wormhole — украдено $320M

Атакующий эксплойтнул уязвимость в валидации подписей на Solana-стороне
моста Wormhole. Украденные средства уже частично конвертированы в ETH.
Третий крупный bridge exploit за последние два месяца — мосты остаются
самым слабым звеном в крипто.

💬 Мосты — ахиллесова пята крипты или просто растущие боли?
Станет ли это последней каплей для мультичейн-подхода?

🔗 ZachXBT

[💬 Обсудить]  ← inline button
```

### Ключевые отличия альфа-поста

- 🚨 маркер перед эмодзи категории — визуальное выделение из ленты
- Авторский саммари с характером и лёгкой иронией (факты точные, подача живая)
- Блок 💬 — провокационный вопрос для дискуссии
- Inline-кнопка «💬 Обсудить» — ведёт в discussion-группу канала

### Тон авторского контента

- Серьёзный по фактам, но с характером
- Лёгкая ирония где уместно
- Без журналистских клише и пафоса
- Стиль: «человек, который разбирается и не боится высказаться»

## Техническая архитектура

### Двухэтапная обработка LLM

1. **Первый вызов (существующий)** — оценка score + обычный summaryRu + titleRu
2. **Второй вызов (новый, только если score ≥ 9)** — editorial-промпт → `editorialSummaryRu` + `discussionQuestion`

Два раздельных вызова а не один, потому что:
- LLM не может знать свой score до написания summary
- Editorial-промпт нужен совершенно другой по стилю
- Второй вызов вызывается редко (score ≥ 9 это 1-3 статьи в день)

### Изменения по модулям

| Модуль | Файл | Изменение |
|--------|------|-----------|
| Types | `src/types/index.ts` | Новые поля: `editorialSummaryRu`, `discussionQuestion` в `LLMFilterResponse` и `StoredArticle` |
| LLM | `src/llm/filter.ts` | Новая функция `generateEditorialContent()` с editorial-промптом |
| Database | `src/database/index.ts` | Колонки `editorial_summary_ru`, `discussion_question` в таблице `articles` |
| Publisher | `src/publisher/index.ts` | Функция `formatAlphaMessage()` + inline keyboard button |
| Orchestrator | `src/orchestrator/index.ts` | После LLM-фильтра: если score ≥ 9, вызвать `generateEditorialContent()` |
| Config | `src/config/index.ts` | `ALPHA_SCORE_THRESHOLD` env variable (default: 9) |

### Inline-кнопка

Используем Telegraf inline keyboard в `sendMessage`. Кнопка «💬 Обсудить» ведёт URL-ом в discussion group. Не требует `bot.launch()` или handling callback queries — это просто URL-кнопка.

## Пререквизиты (ручные)

- Создать Discussion Group и привязать к каналу в настройках Telegram
- Это невозможно сделать через API — только через интерфейс Telegram

## Конфигурация

Новые переменные в `.env`:

```
ALPHA_SCORE_THRESHOLD=9          # Порог для альфа-постов (default: 9)
TELEGRAM_DISCUSSION_GROUP_URL=   # URL discussion-группы канала
```
